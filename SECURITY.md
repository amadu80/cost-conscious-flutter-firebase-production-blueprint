# Security Policy

This repository contains educational guidance and sanitized examples. It does not contain a deployed service.

## Reporting a problem

Do not open a public issue for a vulnerability that could affect a real deployment. Use GitHub private vulnerability reporting when enabled, or contact the maintainer privately through the profile linked to this repository.

Include the affected document, the risk, reproduction conditions, and a suggested mitigation. Never include real credentials, personal data, production identifiers, or exploit data from systems you do not own.

## Scope

Security guidance must be adapted to each application's threat model. No example in this repository is a substitute for a professional security review.
